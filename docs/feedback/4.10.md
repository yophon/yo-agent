## ✅ 补验(2026-07-03 二轮):background 模式真机并发实锤

概念澄清(用户追问「并发和单发是不是都用 subagent_spawn」):**是,只有这一个工具**,并发有两条触发路径——
① 同批多发:模型在同一条响应的 tool_calls 数组放 N 个 spawn(4.10b 内核并发执行);
② `mode:"background"`:发出即返回不阻塞,模型即使逐个调用,子代理也并发跑(4C 既有能力)。
模型嘴里的 `multi_tool_use.parallel` 是 ChatGPT 产品内部封装,我们工具面上不存在,属幻觉。

真机复验路径②:3 个 background spawn(任务=读 README 总结),`/tasks` 面板实拍
**「2 运行中 / 共 3」两个子代理同屏并跑**,收敛耗时 8s/12s/14s、总墙钟 ~20s(串行需 34s+);
面板实时刷新运行态,详情视图完整展示子代理事件流(💬 开场 → ⏺ read → 💬 总结 → 轮结束)。
